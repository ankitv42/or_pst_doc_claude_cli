  ---
  Area 1 — Production Resilience (currently 6.5 → needs 8.5)

  The single biggest gap a Google engineer will probe.

  1.1 Retry + Exponential Backoff on LLM Calls

  What's missing: If Groq returns 429 (rate limit) or 502 (server error), the pipeline immediately fails with no retry.

  What to build:
  # agents/llm_factory.py — wrap every LLM call with tenacity
  from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type

  @retry(
      stop=stop_after_attempt(3),
      wait=wait_exponential(multiplier=1, min=1, max=10),
      retry=retry_if_exception_type((RateLimitError, APIConnectionError)),
  )
  def call_llm_with_retry(llm, messages):
      return llm.invoke(messages)

  Interview answer unlocked: "What happens when Groq is down?" → "We retry 3 times with exponential backoff (1s, 2s, 4s). After 3 failures, the pipeline logs FAILED
  status and the dashboard shows an error. We could also fall back to OpenAI — the llm_factory.py already supports multiple providers, so it's a config change."

  Effort: 2 hours. tenacity is already installed.

  1.2 Fallback LLM Provider

  What's missing: Single provider = single point of failure.

  What to build: In llm_factory.py, add a get_llm_with_fallback() that tries Groq first, then OpenAI if Groq raises after retries.

  Effort: 1 hour. The provider switching logic is already there.

  1.3 Request Timeout on LLM Calls

  What's missing: A hung Groq request blocks a background thread forever.

  What to build:
  ChatGroq(
      model=...,
      api_key=...,
      temperature=0,
      request_timeout=30,  # 30 second hard timeout
  )

  Effort: 15 minutes. Single line per LLM instantiation.

  1.4 API Rate Limiting

  What's missing: Nothing stops 1,000 simultaneous pipeline requests from hitting the API.

  What to build: Add slowapi (1-line FastAPI integration) limiting /api/v1/pipeline/run to 10 requests/minute per IP.

  Effort: 30 minutes.

  Area 2 — Testing Depth (currently 7 → needs 8.5)

  54 unit tests are good. Google wants to see integration tests too.

  2.1 FastAPI Integration Tests (highest value)

  What's missing: Zero tests for the API layer. No test confirms that POST /pipeline/run returns 202, or that POST /pipeline/run with an invalid SKU returns 422.

  What to build: tests/test_api.py using FastAPI's TestClient:
  from fastapi.testclient import TestClient
  from api.main import app

  client = TestClient(app)

  def test_invalid_sku_returns_422():
      resp = client.post("/api/v1/pipeline/run",
                         json={"sku_id": "NONEXISTENT", "store_id": "ST001"})
      assert resp.status_code == 422

  def test_health_returns_200():
      resp = client.get("/health")
      assert resp.status_code == 200

  def test_alerts_returns_list():
      resp = client.get("/api/v1/alerts")
      assert resp.status_code == 200
      assert isinstance(resp.json(), list)

  Effort: 3 hours. No LLM calls — uses TestClient which mocks everything.

  2.2 Agent 3 Formula Compliance Test (crown jewel)

  What's missing: The 54 tests verify the formula in pure Python. They do NOT verify that the LLM actually applies the formula correctly.

  What to build: A single test that calls the real Agent 3 prompt with known synthetic inputs and asserts the output matches expected scores within tolerance:
  # tests/test_agent3_compliance.py
  # Feeds known inputs to the real LLM, checks output is within tolerance
  # Run with: pytest tests/test_agent3_compliance.py --groq-key=...

  This is what Google calls "model evaluation integrated into CI" — the holy grail for AI systems. Even a single test case with known expected output is enough.

  Effort: 3 hours. Needs GROQ_API_KEY at test time (mark as @pytest.mark.integration).
